2-Filtering-Aggregation

Ethan Rogers

Filtering and aggregation of peak-picked and normalized MSI data further concentrate signal and reduce noise. Additionally, these steps reduce the number of features, and therefore the number statistical comparisons. Fewer tests means less stringent multiple hypothesis correction, and greater experimental sensitivity. Unfortuantly feature clustering is computationally intensive and may require access to high performance computing resources.

Non-specific filtering

We will be filtering by mean and standard deviation. We assume that noisy, non-informative features have low variation and low intensity. For simplicity, we will drop features where sample-wide mean intensity and standard deviation are both in the bottom 25th percentile.

Method 1: Using Cardinal core functions and out-of-memory data structures

Time: ~20 minutes

Benefits: keeps large datasets out of memory, simpler to code, easier to understand

Disadvantages: time intensive (can be improved with multiprocessing), can only compute functions with matter implementations (see ?summarizeFeatures for a list of functions)

First, we will bind all the images together, excluding background pixels. This will create a large MSE with all pixels from all samples (minus background) and all features. Different samples are differentiated by run names

mse_all_no_background <- subsetPixels(mse_list_pp_normalized[[1]], tissue_zone!= "background")
for (i in 2:16) {
  mse_all_no_background <- cbind(mse_all_no_background,
                   subsetPixels(mse_list_pp_normalized[[i]], tissue_zone!= "background"))
}

Now we can compute the mean and standard deviation of each feature across all samples. This will take some time as summarizeFeatures is not parallelized by default, and the MSE is large. You can use the BPPARAM argument to speed this up.

mse_all_no_background_summarized <- summarizeFeatures(mse_all_no_background,
                                                      stat = c("mean", "sd"))

Now we can identify features to drop. We will keep features where either the mean or standard deviation is above the 25th percentile.

mz_to_keep_Cardinal_method <- mz(subsetFeatures(mse_all_no_background_summarized, 
                                      !(sd < quantile(sd, p = 0.25) & 
                                          mean < quantile(mean, p = 0.25))))

Method 2: Using data.table

Time: ~30 seconds

Benefits: much faster, can compute any function

Disadvantages: memory intensive even with data.table, complex coding

The code below starts by binding intensity matrices from each sample (along with ROI marker and feature id) into a dataframe. Mean and standard deviation are computed by ROI, and the final output is a list of m/z values to keep, with the same criteria as above.

mz_to_keep_data.table_method <- rbindlist(lapply(seq_along(mse_list_pp_normalized), function(idx) {
  m <- mse_list_pp_normalized[[idx]]
  k <- t(as.matrix(intensity(m)))
  colnames(k) <- paste0("feature_", seq_len(ncol(k)))
  dt <- cbind(as.data.table(pData(m))[, .(x, y, tissue_zone)], k)
  dt_long <- melt(dt, 
                  id.vars = c("x", "y", "tissue_zone"), 
                  variable.name = "feature", 
                  value.name = "intensity")
  dt_long[, feature := sub("feature_", "", feature)
          ][, sample_id := as.character(idx)]
  }), use.names = TRUE)[tissue_zone != "background", .(
    mean = mean(intensity, na.rm = TRUE),
    sd = sd(intensity, na.rm = TRUE)
    ), by = feature
    ][, mz := mz(mse_list_pp_normalized[[1]])
      ][!(mean < quantile(mean, p = 0.25) & sd < quantile(sd, p = 0.25))
        ][, mz]

Feature clustering

Clustering of related features is an active field of research with many implementations. Choosing a clustering algorthim should be done based on what you want to cluster (related molecules, spatial analogs, isotopes, adducts, precursors, etc.) and the methods to cluster (spatial similarity, kendrick mass defect, intenisty ratios, etc.). As such, we treat feature clustering as an interchangable step that takes in MSIs and menaingful feature clusters to the experiment.

As of version 3.6, Cardinal does not offer feature clustering methods. In this work we have used DeepION (ACS Paper, Github) to cluster isotopes. We have also explored mzClustering (Bioinformatics Paper, Original Github, Github of install ready version, made possible by Nihira Golasangi). At the time of publication these methods were experimentally tested but lacked easy to install implementations. mzClustering has been updated by Nihira Golasangi to be install ready from Github. DeepION may require more work to reconcile package conflicts, especially if using Apple M series chipsets. Because theses methods are exist external to this workflow and interchangable with other clustering methodologies, we will prepare the input files for both methods and provide the output of DeepION for clustering.

Both methods use spatial similarity to cluster features. mzClustering creates groups of similar features, while DeepION tests similarity of possible isotopes, as determined by mass differences. To adapt these methods to multi-tissue clustering, we must force the methods to cluster all the samples together. As all samples share the same m/z axis, we can achieve this by stitching together samples’ intensity matrices. Both of these methods are computationally intensive, especially when used to cluster multiple tissue in unison. Additionally, these methods were designed to run on Nvidia GPUs (using CUDA) and must be adapted to run on Apple M series Metal GPU frameworks.

Prepare intensity matrices

First, extract the intensity matrix of each sample, and bind their columns. Both algorithms take a matrix of features and pixels as input - mzClustering requires features \(\times\) pixels while DeepION requires the transpose (pixels \(\times\) features).

k <- NULL
for (n in names(mse_list_pp_normalized)) {
  k <- cbind(k, as.matrix(intensity(mse_list_pp_normalized[[n]])))
}

Each row of a MSE intensity matrix represents a feature, and each column a pixel. Matrix values are intensities. Keep in mind that MSEs are (usually) irregularly shaped, and Cardinal only stores the intensities that are actual sample values. This presents an issue: both DeepION and mzClustering require rectangular images.

We could use sliceImage to get 3D array of \(x \times y \times n_{features}\) - a cube with rectangular slices (features), where the non-sample background of the rectangular slice is NAs - but it would make the images much larger (artificially, mostly NAs) and require more memory.

Instead we will use the fact that, along with a 2D intensity matrix, DeepION and mzClustering take the dimensions of the image as a parameter. Using those dimensions, the algorithms extract the intensity vector of each feature (each row of the intensity matrix, a set of pixel intensities in the order of \((x_1,y_1),(x_2,y_1),...,(x_n,y_n)\)) and reshape the vector using the supplied dimensions. Therefore we shall zero pad the bound intensity arrays, such that each row can be reshaped into a square image. This will disrupt the spatial relationship between pixels - when reshaping each feature image, the new dimensions will be different from the original and there are no sample background pixels - but the spatial relationships between features will be preserved.

# start with the length (# of columns) of the bound intensity matrix
len <- dim(k)[2]
# find the nearest square
new_col <- ceiling(sqrt(len))^2
# find the how many more columns you have to add to get to that square
delta <- new_col - len
# tack on a matrix of 0's to make the square root of the total number of columns the next integer
new_k <- cbind(k, matrix(0, nrow = nrow(k), ncol = delta))

Another caveat is that DeepION and mzClustering cannot handle NA values. As MSIs experience intensity dependent missingness we will zero-fill NA values.

new_k <- replace(new_k, is.na(new_k), 0)
new_k <- as.data.frame(new_k)

Aggregate feature clusters by selecting representative features with the highest mean intensity

As we are aggregating features by selecting the feature with the highest mean intensity to serve as each isotope group’s representative feature, we must computer the feature means across all samples. We can do this in a variety of ways - using either of the above methods, or using matter out of memory data structures themselves.

Now let’s define a function to aggregate isotope groups, keeping only the feature with the highest mean in each group.

reduce_to_candidate_feature_in_group <- function(mse,
                                                 group_frame,
                                                 na.rm = TRUE) {
  # validation checks to ensure the correct columns are present in the group_frame
  stopifnot(
    "group_id" %in% names(group_frame) &
      "mz" %in% names(group_frame) &
      "feature_id" %in% names(group_frame) &
      "full_sample_mean" %in% names(group_frame)
  )

  # check to ensure all mz for grouping_frame and mse
  stopifnot(all(mz(mse)==group_frame$mz))
  
  subsetFeatures(mse, 
                 mz %in% (group_frame %>% 
                            group_by(group_id) %>% 
                            filter(full_sample_mean == max(full_sample_mean)) %>% 
                            ungroup() %>% 
                            pull(mz)))
}

Perform the feature aggregation using the above function.

mse_list_pp_normalized_reduced <- lapply(mse_list_pp_normalized, FUN = function(m) {
  reduce_to_candidate_feature_in_group(m,
                                       iso_merged,
                                       na.rm = TRUE)
  
})
